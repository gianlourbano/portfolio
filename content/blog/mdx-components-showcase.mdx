# MDX Components Showcase <Badge tone="info">tour</Badge>

Instead of skimming a grid of isolated snippets, this page walks through the components in the order you might naturally reach for them while writing a technical post: inline accents, structural callouts, code variants, disclosure blocks, media & file lists, two‑column layouts, and finally math.

<Callout type="info">
    Everything shown here is auto‑registered. Drop these tags directly into any
    post or project MDX file—no manual imports required there.
</Callout>

## 1. Inline Accents

Use `<Kbd>` for key sequences like <Kbd>Ctrl</Kbd> + <Kbd>C</Kbd>. Badges highlight status—<Badge tone="success">Stable</Badge>, <Badge tone="warn">Experimental</Badge>, or <Badge tone="error">Breaking</Badge>. Quick inline metrics fit `<Stat>`: current throughput <Stat label="tok/s" value="3.2k" /> fits neatly in running text without disrupting the flow.

## 2. Notes, Warnings & Context

Short guidance fits inside a gentle `<Note>`: <Note>Use HashRouter for GitHub Pages so deep links work after refresh.</Note> Urgent friction or risk gets a `<Warning>`: <Warning>Never embed secrets; public repo means public content.</Warning> For more emphatic structured info (like success, error, info, warn) swap to `<Callout type="warn">` etc. These boxes create visual rhythm, breaking dense explanations into scannable chunks.

## 3. Code Variants with Tabs

When you want to show the same concept in multiple languages, wrap each variant in a `<Tab>` inside `<Tabs>`. Readers switch context without scrolling past duplicated prose.

## 6. Two‑Column Narrative Layouts

Long form explanations often benefit from a main narrative next to a supporting artifact (diagram, code, metric panel). The `TwoColumn` component lets you specify proportional width and automatic stacking below a breakpoint.

## 7. Math & LaTeX

Inline math like $E = mc^2$ or $\nabla_x \mathcal{L}$ flows naturally in a sentence. For larger expressions, use a display block:

$$
\mathcal{L}(W) = - \sum_{i=1}^N \sum_{c=1}^C y_{ic} \log p_{ic} + \lambda \lVert W \rVert_2^2
$$

Or derive a gradient:

$$
\frac{\partial}{\partial W} (WX - Y)^2 = 2 (WX - Y) X^T
$$

## 8. Putting It Together

Compose these primitives freely: a `TwoColumn` containing a tabbed code block and a figure; a `Details` block inside a `Callout` for optional deep dives; math inline with `Badge`s to flag complexity. The goal is to stay in MDX—no ad‑hoc CSS—while keeping a consistent retro aesthetic.
